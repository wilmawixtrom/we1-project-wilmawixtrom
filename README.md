# Project Summary
I wanted to create a professional and welcoming website for a nail technician business. The goal was to present the services in a clean and elegant way while making the website easy to use for all visitors. I focused on a soft colour scheme, simple navigation, and a modern design that reflects a calm and luxurious salon experience. The website is meant to give customers important information, show available treatments, and make it easy to contact me. 

During this project, I learned a lot. First of all, I learned how quickly time passes when working in front of a computer. You can sit in front of the screen for an hour and only manage to place one image correctly. Of course, this improved throughout the project. The hardest part was at the beginning when I did not fully understand how flexbox worked. Later in the project, I knew much better how to create all the boxes on the remaining pages because I had already created several similar ones before. If I wanted to change something, I knew how to do it, and if I did not know, it did not take as long to learn because I had already understood the concept.

I have also realized how much work goes into every website. A website that looks very simple can still consist of a lot of complicated code. I always want everything to look perfect and clean, and when I saw everyone else’s projects, mine felt very small in comparison. However, I also know that many of them already had experience or had learned programming before. Programming is a very impressive tool that can change and improve a website in really creative ways.

I also learned that I work best under pressure. After the presentations, I got a new sense of motivation. I started looking for more inspiration online and realized that I wanted to improve my website because it felt too flat and boring. I changed almost my entire website over one weekend. Earlier in the project, I thought it was difficult to sit for one or two hours coding, but now I have spent almost two full days working only with code. I am completely exhausted, but also very proud and satisfied with the result.

I never really liked my original header. I mostly used it because it was connected to my business, but it felt very flat and I did not know how to make it fit together with the other pages. It simply did not match the rest of the design. I then decided to download free-to-use images for the headers on all pages because I did not have my own images in the right resolution and size. It did not feel as personal, and I do not know if I would have done the same for a real business website. In that case, I probably would have taken new professional photos myself. Unfortunately, I did not have that possibility during this project. In the end, I became very happy with the result because it became much clearer which page the user had opened when changing between pages, something that was missing before.

I also redesigned my articles and made sure they had better sizes and cleaner shapes. Since the colours on the website were originally based on my old header, the overall theme disappeared a little after changing the images. I decided to switch to a more pink colour theme, while still keeping it professional and not overly feminine.

This project has been very fun, but I wish I had more time. At the same time as this project, I also had to study Physics, Mathematics 4, and write a Komvux project about biodegradable plastic.

Things I did not have enough time to improve on the website but would have liked to:

Remove all validation warnings. I managed to fix all the errors, but not the warnings. I received warnings because I had used h1 several times on the same page. I knew that headings should continue with h2, h3, and so on, but I thought that if several headings were meant to have the same size or take up the same “place” on the page, they should use the same heading tag. I now understand that this is not how it works. I tried to change it, but when I did, several other parts of the website also changed and I did not have enough time to fix everything properly.

- I would have liked to organise my CSS code better. I started doing it but did not have enough time to fully review it before submitting the project.
- I would have liked to make sure that the HTML document had completely correct indentation and spacing before all tags. I also did not have enough time for that.
- I would have liked the website to work perfectly on all screen sizes.
- I would have liked to check if all my images could be opened correctly through GitHub. I tried opening my files from GitHub but could not fully understand how it worked, so I am not completely sure if everything functions properly on another computer besides my own.
- I would have liked to publish the website as a real website online.
- I would have liked to make the form work for real.

As I mentioned earlier, time management was difficult, especially in the beginning when everything felt very hard and slow to work with. It was not very enjoyable at first. However, the project became more and more fun towards the end. At the time of the presentation, I had worked around 17 hours on the project, and today, after working at least seven more hours, I have completed around 29 out of the 30 required hours. I am proud of that, although I probably could have spread those last hours out a bit more evenly. However, I have always worked like that — I am most effective when I work under pressure. As long as I am not too far behind, it usually works out well.
